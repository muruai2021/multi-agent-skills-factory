# 技能设计规格书模板

> 设计师Agent产出物，供开发者编写SKILL.md使用。

---

## 模板

```markdown
# 技能设计规格书

## 基本信息

| 字段 | 内容 |
|------|------|
| 技能名称 | [中文名] |
| skill名称 | skill-xxx |
| 版本 | 1.0.0 |
| 业务域 | [部门/流程] |
| 作者 | [姓名] |
| 创建日期 | [YYYY-MM-DD] |

## 触发条件

**必须满足以下所有条件才触发：**
1. [条件1]
2. [条件2]
3. [条件3]

**满足以下任一条件不触发：**
1. [排除条件1]
2. [排除条件2]

## 核心流程

### 流程1：[子流程名称]

**触发条件**：[什么情况下执行此流程]

**执行步骤**：
1. [步骤1]
2. [步骤2]
3. [步骤3]

**输出**：[此流程的产出物]

### 流程2：[子流程名称]

**触发条件**：[条件]

**执行步骤**：
1. [步骤1]
2. [步骤2]

## 输入输出

| 类型 | 描述 |

| 输出 | [描述输出格式/去向] |

## 边界场景

| 场景 | 处理方式 |

## 目录结构设计

```
skill-xxx/
├── SKILL.md              # 主入口
├── config.yaml           # 运行时配置（可选）
├── metadata.yaml        # 元数据（可选）
└── references/           # 支撑文档
    ├── SOP.md           # 原始SOP
    ├── test_pool.md     # 测试用例
    └── [其他文档]       # 按需添加
```

## 验收标准

1. [标准1]

```

---

## 填写说明

| 字段 | 填写指南 |
|------|----------|
| skill名称 | 英文小写，单词用连字符，如 `skill-order-query` |
| 触发条件 | 必须具体、无歧义，可枚举 |
| 执行步骤 | 每步一个动作，用动词开头 |
| 错误处理 | 覆盖所有可能的异常路径 |
| 边界场景 | 包含极端值、空输入、格式错误等 |
