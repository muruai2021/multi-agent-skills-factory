# 技能自我复盘模板

> 技能每次被调用后，根据输出结果决定是否触发自我复盘。
> 也可由用户手动触发：`/技能名 self-review`

---

## 触发条件（满足任一自动触发）

1. 用户反馈 "不对"、"错了"、"不是这样"
2. 技能执行结果与预期输出明显不符
3. 技能报错或崩溃
4. 主编或评审员要求复盘
5. 手动触发：`/skill-xxx self-review`

---

## 复盘流程

### 第一步：读取错误日志

读取 `references/failure_case_log.md`，确认：
- 是否有新的未修复错误案例
- 是否有同一错误出现 ≥ 3 次（复发性问题）
- 当前错误案例总数

### 第二步：读取本次输入输出

```
触发词：[用户实际输入]
预期输出：[技能描述的预期输出]
实际输出：[技能实际生成的输出]
差异点：[列出具体差异]
```

### 第三步：根因分析

**问题类型分类：**
- 🐛 **Bug** — 技能代码逻辑错误，步骤写错
- 📖 **理解偏差** — 对用户意图理解错误，触发词覆盖不足
- 🛡️ **安全** — 输出涉及敏感信息、注入风险
- 📦 **缺功能** — SOP步骤缺失，边界场景未覆盖
- 🔧 **格式** — 输出格式不符合预期

**根因简述：**
[用一句话描述为什么会出错]

### 第四步：生成改进建议

| 优先级 | 改进措施 | 影响范围 | 预计收益 |
|--------|----------|----------|----------|
| P0 | [立即修复，不能再犯] | 所有类似场景 | 高 |
| P1 | [本迭代内修复] | 当前SOP | 中 |
| P2 | [下个版本规划] | 扩展场景 | 低 |

### 第五步：更新错误日志

将本次错误追加到 `failure_case_log.md`，并更新：
- 错误类型标签
- 根因分析
- 修复建议

## 自我复盘报告模板

```markdown
# 自我复盘报告 — [技能名] — [日期]

## 基本信息
- **技能版本**：vX.X.X
- **复盘触发**：用户反馈 / 报错 / 手动 / 自动（≥5条）
- **错误案例总数**：N 条（未修复：N）

## 本次复盘范围
- 涉及案例：#001, #002, #003
- 问题类型：理解偏差 + 缺功能
- 影响用户数：N（估计）

## 根因分析

### 共性根因
[多个错误案例的共同问题根源]

### 分类统计
| 问题类型 | 数量 | 占比 |
|----------|------|------|
| Bug | N | X% |
| 理解偏差 | N | X% |
| 安全 | N | X% |
| 缺功能 | N | X% |

## 改进计划

### 本次迭代修复（P0 + P1）
1. [具体修改1] → 影响：[范围]
2. [具体修改2] → 影响：[范围]

### 下版本规划（P2）
1. [长期改进项]

## 回归测试计划
修复后，用以下案例回归验证：
- [ ] RE-001：[触发词] — 验证 [预期行为]
- [ ] RE-002：[触发词] — 验证 [预期行为]

## 复盘结论
[总结：技能当前状态，是否可继续使用，还是需要暂停]
```

---

## 复盘后操作

| 结论 | 操作 |
|------|------|
| 严重Bug，影响正确性 | 🚨 立即停止使用，通知主编退回修改 |
| 可接受的边界问题 | ⚠️ 记录到错误日志，下次迭代修复 |
| 无问题，用户误解 | 📝 补充触发词说明，更新示例 |
| 发现新的扩展场景 | 📦 加入 `references/failure_case_log.md` 扩展场景区 |

---

## 与错误日志的协作关系

```
failure_case_log.md          self-review-template.md
（记录每一次错误）                  （分析错误模式）
       ↓                                ↓
  积累 ≥5 条 ──────→ 自动触发复盘流程
                          ↓
                    生成改进计划
                          ↓
                    更新 SKILL.md
                          ↓
                    回归测试验证
                          ↓
                    错误案例移入"已修复"区
```
